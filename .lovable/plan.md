## Что делаем

Большой этап разбит на 4 последовательных коммита внутри одной итерации. Существующая структура и функции сохраняются — добавляем сверху.

### 1. Универсальный конструктор блоков (БД + админка)

Новые таблицы:
- `page_blocks` — `page_key` (about/partners/fabrics/apartment/...), `kind` (banner / text-image / gallery / cards / cta / custom-ref), `title`, `subtitle`, `body`, `image_url`, `gallery` (jsonb массив url), `button_text`, `button_link`, `is_visible`, `sort_order`, `settings` (jsonb для будущих типов).
- Существующие «системные» секции (Преимущества, Шаги, Категории партнёров и т.п.) регистрируются как записи `kind = 'system'` с `system_ref` — их можно скрыть и переставить, но контент по-прежнему редактируется на их специальных экранах.

Админ-экран `/admin/page-blocks/$pageKey`:
- список блоков с drag-and-drop сортировкой (`@dnd-kit`);
- toggle видимости;
- инлайн-редактор полей (заголовок/подзаголовок/текст/кнопка/ссылка);
- загрузка одного фото и галереи в bucket `page-blocks` (создаём публичный);
- кнопка «Добавить блок» с выбором типа.

Страницы About / Partners / Fabrics / Apartment получают рендерер `<PageBlocks pageKey="about" systemSlots={{ advantages: <Advantages/>, ... }} />`, который читает порядок из БД и подставляет либо системный слот, либо универсальный блок.

### 2. Верхние баннеры страниц

Отдельный `kind = 'hero-banner'` блок (по одному закреплённому сверху на страницу). Поля: заголовок, текст, кнопка+ссылка, фоновое изображение, контраст текста. Редактируется в том же `/admin/page-blocks/$pageKey` отдельной секцией «Баннер».

### 3. Главный слайдер вместо текущего hero

Новая таблица `home_slides`: `title`, `subtitle`, `button_text`, `button_link`, `image_url` (мебель), `bg_color`, `text_align`, `is_visible`, `sort_order`. Сидируем 5 слайдов из ТЗ.

Компонент `<HeroSlider />` (embla-carousel-react, уже в проекте):
- автопрокрутка 6с, пауза при hover;
- плавный crossfade, текст слева с анимацией fade-up, фото справа с лёгким scale-in;
- точки + стрелки;
- использует существующие токены дизайна.

Заменяет текущий hero в `src/routes/index.tsx`. Админ-экран `/admin/home-slides` со списком, drag-and-drop, загрузкой фото.

### 4. Лёгкий рефреш UX

- Подключаем `useReveal` (уже есть) к карточкам товаров, секциям About/Partners/Fabrics/Apartment — мягкое fade-up при скролле.
- Унифицируем `ProductCard`: одинаковые отступы/радиусы/тени через токены (без переработки разметки).
- Lightbox в галереях («Мебель у наших покупателей», галерея партнёров, галерея About) на базе уже установленного `react-image-lightbox`-эквивалента — если его нет, простая модалка с zoom.
- Hover-эффекты у кнопок и изображений (transition transform/shadow).
- Адаптивные правки только там, где сетка ломается на мобилке (карточки тканей и партнёров).

### Технические детали

- Storage bucket `page-blocks` (public read, authenticated write).
- RLS: чтение всех `page_blocks` / `home_slides` всем; запись — только `admin` через `has_role`.
- GRANT SELECT для `anon` и `authenticated`, GRANT ALL для `service_role` + admin write через policy.
- Drag-and-drop: уже установленный `@dnd-kit/sortable` (если нет — ставим).
- Embla carousel уже есть в зависимостях shadcn.
- Существующие админ-экраны (about, partners content, apartment, etc.) остаются — конструктор работает параллельно.

### Что НЕ трогаем

- Каталог, корзина, избранное, формы, заявки, интеграции, апартаменты-калькулятор — без изменений.
- Текущий дизайн карточек и страниц — только унификация токенов и анимации, без перерисовки.
- Структуру роутов — все новые админ-страницы под `_authenticated/admin.*`.

### Порядок выполнения

1. Миграция БД + storage bucket + сиды слайдов и регистрация системных блоков для 4 страниц.
2. Админка: `/admin/home-slides`, `/admin/page-blocks/$pageKey`, пункты в сайдбаре.
3. Фронт: `HeroSlider` + рендерер `PageBlocks` + интеграция в About/Partners/Fabrics/Apartment.
4. Рефреш: reveal-анимации, унификация карточек, lightbox в галереях.

После завершения владелец сможет управлять баннерами, слайдером, порядком и видимостью блоков, текстами/фото/галереями всех 4 страниц через админку без участия разработчика.
