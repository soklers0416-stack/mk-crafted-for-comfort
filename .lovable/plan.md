# План доработки каталога

Дизайн, цвета, шрифты и главная страница не меняются. Все правки — в данных, карточке товара и странице товара.

## 1. Структура данных (`src/lib/data.ts`)

Расширяю тип `Product`, чтобы покрыть все поля из ТЗ и сделать структуру предсказуемой для ручного добавления:

```ts
type Product = {
  id: string;
  category: string;          // slug категории
  title: string;
  price: number;
  priceFrom?: boolean;
  description: string;

  // Галерея — до 5 отдельных полей (удобно редактировать через Visual Editor)
  photo1: string;            // обязательное главное фото
  photo2?: string;
  photo3?: string;
  photo4?: string;
  photo5?: string;

  // Характеристики (отдельные поля для удобства редактирования)
  sizes?: SizeRow[];
  sleepingPlace?: string;    // спальное место
  mechanism?: string;        // механизм
  filling?: string;          // наполнение
  hasBox?: boolean;          // короб: да/нет
  availability?: "в наличии" | "под заказ";
  productionTime?: string;   // срок изготовления
  specs?: Spec[];            // доп. характеристики (опционально)

  // Акция
  sale?: { enabled: boolean; label?: string; oldPrice?: number; newPrice?: number; text?: string };
};
```

В начале файла оставлю подробный комментарий-инструкцию «как добавить товар / как заменить фото / как включить акцию» — чтобы владелец мог работать без AI.

Существующие товары переведу на новый формат (photo1..photo5 вместо image+gallery; sleepingPlace/mechanism/filling/hasBox/productionTime вынесу из массива specs в отдельные поля). Старые поля `image`/`gallery` уберу — вместо них вспомогательный геттер `getGallery(p)` соберёт массив из photo1..photo5, отфильтровав пустые.

## 2. Карточка товара в каталоге (`src/components/ProductCard.tsx`)

Содержимое без изменения дизайна, но:

- Картинка — `<img>` с `object-contain` на белом фоне (`bg-surface-muted`), без обрезки. Соотношение блока сохраняю прежнее (`aspect-[5/4]`), фото вписывается целиком.
- Показываю: главное фото (photo1), название, цену, 2–3 кратких характеристики (спальное место / механизм / наличие), кнопки «Подробнее» и «Оставить заявку» (вместо текущей «В корзину» — открывает `RequestDialog`).
- Плашка «Акция» (`sale.enabled`) остаётся в левом верхнем углу.

## 3. Страница товара (`src/routes/product.$id.tsx`)

- Галерея до 5 фото: главный `<img>` с `object-contain` + миниатюры (тоже `object-contain`). Никаких `background-image`.
- Блок информации: полное описание, цена (с акцией), отдельные строки «Размеры / Спальное место / Механизм / Наполнение / Короб / Наличие / Срок изготовления» — рендерятся из новых полей `Product`. Таблица размеров и блок «Характеристики» остаются как сейчас, если поля заполнены.
- Кнопки: «Оформить заказ», «Я просто спросить», «Рассчитать доставку», «Рассрочка» (новая — открывает `RequestDialog` с темой «Рассрочка»), «Посмотреть в другой расцветке» (уже есть).
- Похожие товары снизу — оставляю как есть.

## 4. Visual Editor совместимость

Каждое фото в карточке и в галерее — обычный `<img src={...} />` с прямой ссылкой на импортированный ассет. Это позволяет менять фото через Visual Editor по клику на конкретное изображение, без AI и без `background-image`.

## 5. Что НЕ трогаю

- Главная страница, шапка, подвал, цвета, шрифты, стиль.
- Категории, страница `/catalog`, корзина, акции `/promotions`, формы.

## Технические детали

- Файлы для правок: `src/lib/data.ts`, `src/components/ProductCard.tsx`, `src/routes/product.$id.tsx`. Возможны мелкие правки в `src/routes/index.tsx` и `src/routes/catalog.tsx`, если они обращаются к `product.image`/`product.gallery` — заменю на `product.photo1` / `getGallery(product)`.
- `object-contain` + фиксированное соотношение блока (`aspect-[5/4]` в карточке, `aspect-[4/3]` в галерее товара) гарантирует, что фото видно целиком и сетка карточек не «прыгает».
- Инструкция по добавлению товара — комментарий в `data.ts` с готовым шаблоном объекта.
