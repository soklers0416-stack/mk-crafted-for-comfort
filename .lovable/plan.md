## Доработка сайта МК Мебель

Большой объём работы. Разобью на этапы, каждый завершается рабочим функционалом в админке и на сайте.

---

### Этап 1. Раздел «Ткани» (база + админка + страница)

**База данных:**
- `fabric_categories` — id, slug, title, sort_order
- `fabrics` — id, code, title, category_slug, description, characteristics (jsonb: для детей, для животных, лёгкость ухода, износостойкость, особенности), recommendations, surcharge (numeric), sample_photo (фото образца), furniture_photos (jsonb массив URL), sort_order
- `product_fabrics` — связь товар↔ткань (many-to-many)
- RLS: public read, admin write через `has_role`
- GRANT для anon/authenticated/service_role

**Первоначальное наполнение:** scraping референса `материалы-мебель.рф/materials/` через Firecrawl, импорт категорий, тканей и фото в БД.

**Страница `/fabrics`:**
- каталог с фильтром по категориям и поиском по названию/коду
- карточки тканей с фото
- детальная страница ткани `/fabrics/$id` — описание, характеристики, рекомендации, образец + реальные фото мебели

**Админка `/admin/fabrics`:**
- список, создание, редактирование, удаление тканей
- загрузка фото (через прокси `/api/public/photo/$`)
- управление категориями тканей

**В редакторе товара:** блок «Доступные ткани» — чекбоксы.

---

### Этап 2. Выбор ткани в карточке товара

- Блок «Выбранная ткань» с миниатюрой и кнопкой «Изменить ткань»
- Боковая Sheet-панель: фильтр, поиск, список доступных тканей, описание/характеристики, кнопка «Выбрать»
- Доплата прибавляется к цене товара
- Выбранная ткань сохраняется в корзине и передаётся в заявку
- После выбора — блок «Хотите увидеть мебель в этой ткани?» с формой (телефон → заявка с названием товара, тканью, телефоном)

---

### Этап 3. Переработка «О компании» с полной редактируемостью

**База данных:**
- `about_content` (key/value jsonb) — все тексты, заголовки, кнопки первого экрана, блока «почему ниже цены», блока «почему выбирают нас», шоурум-блока
- `about_advantages` — преимущества (icon, title, description, sort_order)
- `about_stats` — счётчики (label, value, sort_order)
- `about_steps` — этапы «как работаем» (title, description, sort_order)
- `customer_photos` — фото у покупателей (photo, city, model, comment, sort_order)
- `gallery_items` — слайдер (photo, category: производство/шоурум/мебель/доставка/сборка, sort_order)
- `faqs` — вопрос/ответ/порядок

**Страница `/about`** перестраивается под все блоки выше: hero, преимущества, «почему цены ниже», «почему нас выбирают», цифры, шоурум + кнопка «Построить маршрут» (Яндекс/Google maps), 4 шага, галерея клиентов (lightbox при клике), слайдер галереи, FAQ (Accordion), форма консультации (имя + телефон → requests).

**Админка:**
- `/admin/about` — общие тексты, преимущества, статистика, шаги, шоурум
- `/admin/customer-photos` — фото клиентов
- `/admin/gallery` — слайдер
- `/admin/faqs` — FAQ

---

### Этап 4. Финальная проверка

- Все данные редактируются из админки без правки кода
- Загрузка изображений через существующий прокси `/api/public/photo/$`
- Изменения отображаются сразу (invalidateQueries)

---

### Технические детали

- Stack: TanStack Start, Supabase, React Query, Tailwind, shadcn/ui (Sheet, Accordion, Dialog, Tabs)
- Все таблицы создаются миграцией с GRANT + RLS
- Скрапинг тканей-референса однократным server-fn-скриптом, результат пишется в БД
- Хранилище фото: существующий bucket `product-photos` + прокси

---

### Объём и порядок

Это 3 крупных независимых этапа. Предлагаю делать **последовательно** — после каждого этапа вы проверяете и подтверждаете. Начну с **Этапа 1 (Ткани)**, так как от него зависит Этап 2.

**Вопросы перед стартом:**

1. Подтверждаете ли порядок: сначала Этап 1 (Ткани + админка), потом Этап 2 (выбор ткани в товаре), потом Этап 3 (О компании)?
2. Для первоначального наполнения тканей: парсить референс автоматически через Firecrawl (может быть неполным/неточным) или вы предоставите CSV/список вручную?
3. Для блока «Построить маршрут» в шоуруме — какой адрес и куда вести (Яндекс.Карты / Google Maps)?
